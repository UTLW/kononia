## 📋 STEP 4 — TRPC ROUTER STRUCTURE

The Express server runs tRPC. The router lives at `file apps/server/src/router.ts`.

### Main Router: `appRouter`

```typescript
// All procedure types are re-exported from packages/api
export const appRouter = t.router({
  meals: t.router({
    list: publicProcedure
      .input(z.object({
        cuisine: z.string().optional(),
        fastingType: z.string().optional(),
        limit: z.number().default(50),
        offset: z.number().default(0),
      }))
      .query(async ({ input }) => { /* return paginated meals */ }),
    get: publicProcedure
      .input(z.object({ id: z.string() }))
      .query(async ({ input }) => { /* return full meal with ingredients + steps */ }),
    getSnacks: publicProcedure
      .input(z.object({ cuisine: z.string().optional(), fastingType: z.string().optional() }))
      .query(async ({ input }) => { /* return snacks */ }),
    search: publicProcedure
      .input(z.object({ query: z.string() }))
      .query(async ({ input }) => { /* search meals by name */ }),
  }),

  // ── AUTHENTICATED (Better-Auth session required) ────
  user: t.router({
    getProfile: protectedProcedure
      .query(async ({ ctx }) => { return ctx.db.select().from(users).where(eq(users.id, ctx.session.user.id)) }),
    updatePlan: protectedProcedure
      .input(z.object({ plan: z.enum(["free", "annual"]) }))
      .mutation(async ({ ctx, input }) => { /* update user plan */ }),
  }),
});
```

## 📋 STEP 7 — POLAR PAYMENTS

**Flow:**

1. User clicks "Upgrade to Annual" in settings
2. Redirect to Polar checkout (one-time $9.99/year)
3. Polar webhook → server updates user's `plan` to "annual"
4. Frontend checks plan on page load → unlocks features

**Polar Events to Handle:**

- `order.completed` → set plan = "annual"
- `subscription.updated` → update plan status
- `subscription.cancelled` → set plan = "free"
